                                                                  平台简介

                                                                  蓝莓数据工厂是一个低代码可视化数据编排平台，面向需要跨系统、跨数据库进行数据处理与集成场景的用户。

                                                                  核心能力

                                                                  • 可视化编排：通过拖拽节点和连线的方式构建数据处理流程（DAG），无需编写代码，所见即所得

                                                                  • 多数据源支持：统一管理 MySQL、MongoDB 等数据库连接，以及外部 HTTP API 接口

                                                                  • 条件分支：支持 IF 条件判断节点，根据数据内容动态选择不同的执行路径

                                                                  • 变量传递：节点之间自动传递数据，支持 {{节点.字段}} 语法引用上游结果

                                                                  • 定时调度：支持 Cron 表达式定时执行任务，可按分钟、小时、天、周、月灵活配置

                                                                  • 开放接口：将编排好的任务一键暴露为 HTTP 接口，供外部系统直接调用，支持调用配额管控

                                                                  • 环境隔离：支持 DEV → TEST → PROD 三套环境，资源通过晋升机制流转，版本可追溯、可回滚

                                                                  • AI 助手：内置蓝莓智能助手，可随时询问数据库结构、表字段等问题

                                                                  4. 开放平台

                                                                  功能说明

                                                                  将已创建的任务暴露为对外开放的 HTTP 接口，供外部系统直接调用。

                                                                  操作说明

                                                                  1. 注册接口：选择一个已启用的任务，设置接口路径、请求方法（GET / POST）、每日调用限额

                                                                  2. 启用 / 禁用：控制接口是否对外开放

                                                                  3. 测试调用：在页面内直接测试接口是否正常工作

                                                                  4. 每日配额：可限制每个接口每天的最大调用次数，防止滥用

                                                                  调用方式

                                                                  注册后的接口可通过以下地址调用：

                                                                  POST /open/{环境}/{路径}

                                                                  例如：POST /open/DEV/user-query

                                                                  外部调用时会在请求体中传入参数，这些参数在任务中通过 {{params.xxx}} 引用。

                                                                  5. 日志管理

                                                                  功能说明

                                                                  记录每次任务执行的结果，包括执行状态、耗时、每个节点的执行详情。

                                                                  操作说明

                                                                  1. 按环境筛选日志

                                                                  2. 查看每条日志的：任务 ID、执行状态（成功 / 失败）、开始时间、结束时间、耗时

                                                                  3. 点击"详情"查看每个节点的执行结果

                                                                  4. 管理日志：批量选择并删除日志

                                                                  5. 清理历史：按天数批量清理过期日志

                                                                  6. 环境管理与版本控制

                                                                  环境说明

                                                                  平台支持三套环境：

                                                                  晋升流程

                                                                  DEV（开发） ──晋升──▶ TEST（测试） ──晋升──▶ PROD（生产）

                                                                  • 晋升只能按顺序进行：DEV → TEST → PROD，不能跳级

                                                                  • 晋升时会在目标环境创建或更新同名资源

                                                                  • 每次晋升会生成一个版本快照

                                                                  版本管理

                                                                  1. 创建新版本：在 DEV 环境创建当前配置的快照

                                                                  2. 查看快照：查看历史版本的完整配置（JSON 格式）

                                                                  3. 回滚：将配置恢复到某个历史版本

                                                                  4. 版本历史：时间线展示所有版本变更记录

                                                                  7. 蓝莓数据助手

                                                                  功能说明

                                                                  页面左下角的蓝莓🫐图标是 AI 智能助手，可以回答关于数据库表结构、数据查询等的问题。

                                                                  使用方式

                                                                  1. 点击左下角蓝色鲸鱼图标打开聊天窗口

                                                                  2. 输入问题，如"有哪些数据表"、"users 表有哪些字段"

                                                                  3. 助手会流式返回答案，支持 Markdown 格式（代码块、表格等）

                                                                  4. 可点击垃圾桶图标清空对话历史

                                                                  8. 常见使用场景

                                                                  场景一：定时同步 MySQL 数据到 MongoDB

                                                                  1. 在 数据库管理 中添加 MySQL 和 MongoDB 数据源

                                                                  2. 在 任务管理 中新建任务

                                                                  3. 拖入 START → MySQL → MongoDB → END

                                                                  4. MySQL 节点写查询 SQL

                                                                  5. MongoDB 节点选择 insertOne 操作，document 引用 MySQL 的查询结果

                                                                  6. 设置定时调度（如每天 08:00）

                                                                  7. 保存并启用

                                                                  场景二：根据条件调用不同接口

                                                                  1. 拖入 START → MySQL → IF → 两个 API → END

                                                                  2. MySQL 节点查询数据

                                                                  3. IF 节点设置条件（如 {{mysql_01.count}} > 10）

                                                                  4. true 分支连接 API-A，false 分支连接 API-B

                                                                  5. 保存并执行

                                                                  场景三：开放接口供第三方调用

                                                                  1. 创建并测试好任务

                                                                  2. 在 开放平台 注册接口，关联该任务

                                                                  3. 第三方系统通过 POST /open/DEV/接口路径 调用

                                                                  4. 在日志管理中查看调用记录

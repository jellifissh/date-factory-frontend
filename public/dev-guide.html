                                      • 后续组件开发指南

                                      本文档介绍如何在任务编排器中扩展新的 DAG 节点类型。

                                      1. 整体架构

                                      任务编排采用 DSL 驱动 + 策略模式：

                                      前端（TaskEditor.vue）                    后端（task-service）
                                      ┌─────────────────┐                      ┌──────────────────────┐
                                      │  左侧：节点面板  │  ── DSL JSON ──▶     │  DagEngine           │
                                      │  中间：X6 画布   │   (nodes + edges)    │    ├─ 拓扑排序 (Kahn) │
                                      │  右侧：属性面板  │                       │    ├─ 分层并行执行     │
                                      └─────────────────┘                      │    └─ strategyMap     │
                                                                                │        ├─ MysqlStrategy │
                                      GET /task/node-types                      │        ├─ MongoStrategy │
                                      （自动返回所有策略的入参/出参定义）             │        ├─ IfStrategy    │
                                                                                │        ├─ ApiStrategy   │
                                                                                │        └─ ...           │
                                                                                └──────────────────────┘

                                      核心设计：每个节点类型对应一个 NodeStrategy 实现类，标记 @Component 后由 Spring 自动发现并注册到 DagEngine.strategyMap。新增节点类型无需修改任何已有代码。

                                      2. 后端开发

                                      2.1 核心接口

                                      所有节点策略实现 NodeStrategy 接口：

                                      // 路径: com.datafactory.task.dag.NodeStrategy
                                      public interface NodeStrategy {
                                      

                                          /** 执行节点逻辑 */
                                          NodeOutput execute(NodeInput input, TaskContext context);
                                      

                                          /** 节点类型标识，如 "PYTHON"、"FILTER" */
                                          String getType();
                                      

                                          /** 入参定义（供前端编辑器渲染配置表单） */
                                          default List<ParamDef> getInputDefs() { return List.of(); }
                                      

                                          /** 出参定义（供下游节点选择变量） */
                                          default List<ParamDef> getOutputDefs() { return List.of(); }
                                      

                                          /** 按子操作获取入参定义（用于动态表单，如 MongoDB 的不同操作） */
                                          default List<ParamDef> getInputDefs(String operation) { return getInputDefs(); }
                                      

                                          /** 按子操作获取出参定义 */
                                          default List<ParamDef> getOutputDefs(String operation) { return getOutputDefs(); }
                                      

                                          /** 支持的子操作列表（空列表表示无子操作） */
                                          default List<String> getOperations() { return List.of(); }
                                      }

                                      2.5 完整示例：Python 节点

                                      package com.datafactory.task.dag;
                                      

                                      import lombok.extern.slf4j.Slf4j;
                                      import org.springframework.stereotype.Component;
                                      import org.springframework.web.client.RestTemplate;
                                      

                                      import java.util.*;
                                      

                                      @Slf4j
                                      @Component
                                      public class PythonStrategy implements NodeStrategy {
                                      

                                          private final RestTemplate restTemplate = new RestTemplate();
                                      

                                          @Override
                                          public String getType() {
                                              return "PYTHON";
                                          }
                                      

                                          @Override
                                          public NodeOutput execute(NodeInput input, TaskContext context) {
                                              // 1. 从 config 中读取参数
                                              String script = input.getConfigAsString("script");
                                              if (script == null || script.isBlank()) {
                                                  return NodeOutput.fail("缺少必填参数: script");
                                              }
                                      

                                              // 2. 变量替换（支持引用上游节点输出）
                                              script = context.resolve(script);
                                      

                                              // 3. 执行逻辑
                                              try {
                                                  Map<String, Object> requestBody = new HashMap<>();
                                                  requestBody.put("script", script);
                                      

                                                  // 示例：调用外部 Python 执行服务
                                                  @SuppressWarnings("unchecked")
                                                  Map<String, Object> response = restTemplate.postForObject(
                                                      "http://python-executor:5000/execute",
                                                      requestBody, Map.class
                                                  );
                                      

                                                  // 4. 构建输出并写入上下文
                                                  Map<String, Object> data = new LinkedHashMap<>();
                                                  data.put("stdout", response.get("stdout"));
                                                  data.put("stderr", response.get("stderr"));
                                                  data.put("exitCode", response.get("exitCode"));
                                      

                                                  context.put(input.nodeId(), data);
                                                  log.info("PYTHON 节点 [{}] 执行完成", input.nodeId());
                                                  return NodeOutput.ok(data);
                                      

                                              } catch (Exception e) {
                                                  return NodeOutput.fail("Python 执行失败: " + e.getMessage());
                                              }
                                          }
                                      

                                          @Override
                                          public List<ParamDef> getInputDefs() {
                                              return List.of(
                                                  new ParamDef("script", "string", "Python 脚本", true)
                                              );
                                          }
                                      

                                          @Override
                                          public List<ParamDef> getOutputDefs() {
                                              return List.of(
                                                  new ParamDef("stdout", "string", "标准输出", false),
                                                  new ParamDef("stderr", "string", "错误输出", false),
                                                  new ParamDef("exitCode", "number", "退出码", false)
                                              );
                                          }
                                      }

                                      @Slf4j
                                      @Component
                                      public class FilterStrategy implements NodeStrategy {
                                      

                                          private static final List<String> OPERATIONS = List.of(

                                          @Override

                                          public List<String> getOperations() { return OPERATIONS; }

                                              String operation = input.getConfigAsString("operation");
                                              String sourceData = input.getConfigAsString("sourceData");
                                      

                                              // 变量替换：sourceData 可能是 {{mysql_01.result}}
                                              sourceData = context.resolve(sourceData);
                                      

                                              // ... 根据 operation 执行不同的过滤逻辑 ...
                                      

                                              Map<String, Object> data = new LinkedHashMap<>();
                                              data.put("result", filteredResult);
                                              data.put("count", count);
                                              context.put(input.nodeId(), data);
                                              return NodeOutput.ok(data);
                                          }
                                      

                                          @Override
                                          public List<ParamDef> getInputDefs() {
                                              return List.of(
                                                  new ParamDef("sourceData", "string", "数据来源", true)
                                              );
                                          }
                                      

                                          // 根据子操作返回不同的入参定义
                                          @Override
                                          public List<ParamDef> getInputDefs(String operation) {
                                              List<ParamDef> base = new ArrayList<>(getInputDefs());
                                              return switch (operation) {
                                                  case "selectFields" -> { base.add(new ParamDef("fields", "string", "字段列表(逗号分隔)", true)); yield base; }
                                                  case "where"        -> { base.add(new ParamDef("condition", "string", "过滤条件", true)); yield base; }
                                                  case "limit"        -> { base.add(new ParamDef("count", "number", "限制条数", true)); yield base; }
                                                  case "sort"         -> { base.add(new ParamDef("field", "string", "排序字段", true)); yield base; }
                                                  default -> base;
                                              };
                                          }
                                      

                                          @Override
                                          public List<ParamDef> getOutputDefs() {
                                              return List.of(
                                                  new ParamDef("result", "json", "过滤结果", false),
                                                  new ParamDef("count", "number", "结果行数", false)
                                              );
                                          }
                                      }

                                      3.3 第三步：出参配置（可选）

                                      右侧面板底部有出参映射区域，通过 nodeTypeDefsMap 自动从后端 /task/node-types 获取出参定义，无需前端额外配置。

                                      4. DSL 数据结构

                                      前端画布保存到数据库的 DSL 格式：

                                      {
                                        "nodes": [
                                          {
                                            "id": "start_1",
                                            "type": "START",
                                            "x": 100, "y": 50,
                                            "outputs": [
                                              { "name": "myParam", "path": "{{params.keyword}}" }
                                            ]
                                          },
                                          {
                                            "id": "mysql_1",
                                            "type": "MYSQL",
                                            "x": 300, "y": 50,
                                            "dbId": 1,
                                            "sql": "SELECT * FROM users WHERE name = '{{start_1.myParam}}'",
                                            "assertCondition": "{{mysql_1.count}} > 0",
                                            "outputs": [
                                              { "name": "firstUserName", "path": "result[0].name" }
                                            ]
                                          },
                                          {
                                            "id": "python_1",
                                            "type": "PYTHON",
                                            "x": 500, "y": 50,
                                            "script": "print('{{mysql_1.firstUserName}}')"
                                          },
                                          {
                                            "id": "end_1",
                                            "type": "END",
                                            "x": 700, "y": 50
                                          }
                                        ],
                                        "edges": [
                                          { "source": "start_1", "target": "mysql_1" },
                                          { "source": "mysql_1", "target": "python_1" },
                                          { "source": "python_1", "target": "end_1" }
                                        ]
                                      }

                                      DagEngine 处理流程：

                                      1. 解析 nodes + edges

                                      2. Kahn 算法拓扑分层排序

                                      3. 按层顺序执行，同层节点并行（CompletableFuture）

                                      4. 每个节点从 strategyMap 查找对应策略并执行

                                      5. IF 节点根据条件结果标记跳过分支

                                      6. 所有节点结果存入 TaskContext，支持 {{nodeId.field}} 变量引用

                                      6. 开发检查清单

                                      新增一个节点类型，按以下清单逐项确认：

                                      后端

                                      • 在 com.datafactory.task.dag 包下创建新的 XxxStrategy.java

                                      • 类标注 @Component 和 @Slf4j（需要日志时）

                                      • 实现 getType() 返回唯一类型标识（如 "PYTHON"）

                                      • 实现 execute(NodeInput, TaskContext) 编写执行逻辑

                                      • 实现 getInputDefs() 定义入参（前端表单渲染依赖此定义）

                                      • 实现 getOutputDefs() 定义出参（下游变量引用依赖此定义）

                                      • 如有子操作，实现 getOperations() 和 getInputDefs(operation) / getOutputDefs(operation)

                                      • 执行成功后调用 context.put(input.nodeId(), data) 存储输出

                                      • 需要 Feign 调用其他服务时，注入对应的 FeignClient

                                      • 字符串参数中使用 context.resolve() 进行变量替换

                                      前端

                                      • 在 nodeTypes 数组中添加 { type, label, color }

                                      • 在 getNodeColor() 中同步添加颜色映射

                                      • 在右侧属性面板模板中添加对应 v-if="selectedNode.type === 'XXX'" 的配置表单

                                      • 如有子操作，添加操作下拉和动态表单

                                      • 测试：拖拽新节点到画布 → 配置属性 → 连线 → 保存 → 执行 → 验证输出

                                      无需修改的部分

                                      • DagEngine — 自动通过 Spring 发现新策略

                                      • /task/node-types API — 自动返回所有策略的定义

                                      • 拓扑排序、并行执行、IF 分支 — 通用能力自动复用

                                      • 变量替换（{{...}}）— TaskContext 通用能力

                                      • 断言检查（assertCondition）— DagEngine 通用能力
